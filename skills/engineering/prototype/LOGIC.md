# 逻辑 Prototype

用于验证状态模型、业务规则、数据流或边界条件。

## 形态

构建一个很小的终端程序：

- 可以交互式选择操作；
- 打印当前状态；
- 包含几个预置场景；
- 允许快速重置；
- 不依赖生产数据库。

## 适用问题

- 状态机是否覆盖所有转移？
- 业务规则冲突时优先级是什么？
- 数据模型是否让常见操作变简单？
- 边界情况是否会产生奇怪状态？
- 某个流程是否需要额外接缝？

## 实施规则

- 放在相关模块旁边，名称包含 `prototype` 或 `scratch`。
- 顶部写清楚：问题、运行命令、删除条件。
- 复用真实领域类型和纯函数，但不要修改生产接口来服务 prototype。
- 可以复制少量代码以换取速度；这是一次性代码。
- 输出应帮助人判断模型是否合理，而不是模拟完整产品。

## 输出示例

```text
当前状态：Cart(items=2, coupon=SUMMER)
可选操作：
1. add item
2. remove item
3. apply coupon
4. checkout
```

## 完成汇报

- 运行命令；
- 覆盖的场景；
- 观察到的设计风险；
- 建议保留、调整或删除的模型。
