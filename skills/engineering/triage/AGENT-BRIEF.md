# Agent Brief

Agent brief 是给另一个 agent 独立执行 issue 的说明。它必须具体、可验证、少依赖上下文。

## 模板

```markdown
## 背景

这个问题为什么存在？用户或系统受什么影响？

## 目标

实现或修复什么可观察行为？

## 非目标

明确不做什么，避免 agent 扩大范围。

## 相关上下文

- 代码区域：...
- 领域术语：...
- ADR：...
- 相关 issue/PR：...

## 验收标准

- [ ] ...
- [ ] ...

## 测试/验证

运行：

```bash
...
```

## 风险

- ...
```

## 规则

- 不要只写“修复 bug”。
- 不要要求 agent 自己做产品决策。
- 不要省略验证命令。
- 如果有未知问题，保持 `needs-info`，不要标记 ready。
- 使用项目领域词汇。
