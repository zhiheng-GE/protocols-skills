# 仅对硬依赖技能显式提示 `/setup-matt-pocock-skills`

工程类 skills 依赖每个仓库自己的配置：问题追踪器、分流标签词汇、领域文档布局。这些配置由 `/setup-matt-pocock-skills` 初始化。部分技能没有这些配置就无法正确工作；另一些技能只是利用这些配置提升输出质量，即使没有也可以降级运行。

因此我们将技能分为 **硬依赖** 和 **软依赖**：

- **硬依赖**（`to-issues`、`to-prd`、`triage`）：必须包含明确提示：“如果尚未提供这些配置，请运行 `/setup-matt-pocock-skills`。” 没有映射时，输出会写错位置或打错标签，不只是表达不够精准。
- **软依赖**（`diagnose`、`tdd`、`improve-codebase-architecture`、`zoom-out`）：只在正文中笼统引用“项目领域词汇表”和“相关 ADR”。如果文档不存在，技能仍然可用，只是输出不够贴合项目语言。

这样做可以让软依赖技能保持轻量，避免把 setup 提示复制到并不关键的位置。
